# Loan Default Prediction

End-to-end binary-classification project: given a loan application, predict the probability of default. Three models (logistic regression, random forest, tuned LightGBM) compared on PR-AUC, with cost-aware threshold selection (FN:FP = 5:1) and SHAP interpretability against a known synthetic data-generating process.

The pipeline is built on a synthetic Lending-Club-style dataset for reproducibility, with a single swap point (`src/data.load_data`) so a real Lending Club CSV dropped into `data/raw/` runs through the same code.

**Headline result.** Tuned LightGBM hits PR-AUC 0.74 / ROC-AUC 0.92 on the held-out test set; moving from the default 0.5 threshold to the cost-optimal 0.14 cuts expected loss by **33.9%** under the assumed 5:1 FN:FP ratio. Full numbers and discussion in [reports/report.md](reports/report.md).

## Quickstart

```bash
make install         # creates .venv, installs pinned deps
make test            # 56 tests
make smoke           # 5k rows + 10 tuning iters, ~90s — sanity check
make train           # full run with random split (~7 min), saves model + predictions
make train-temporal  # OOT comparison (~30s, reuses tuned hyperparameters)
make notebooks       # execute all seven notebooks in order, in-place
```

`make all` chains install → test → train → notebooks. `make verify` wipes generated state and reproduces from scratch. `make clean` removes only generated state. `make help` lists targets.

## Using real Lending Club data

The project is synthetic by default. To run the same pipeline on a real Lending Club CSV:

1. Download the accepted-loans CSV from Lending Club (Kaggle mirror works) and drop it in `data/raw/`. Any filename ending in `.csv` is fine; the loader picks the first match.
2. In any notebook or script: `df = load_data(source="lendingclub")`. Returns the same canonical schema (17 columns) as `load_data(source="synthetic")`.

`src/data.py::_load_lendingclub` handles the LC quirks:

| LC column | Internal | Notes |
|---|---|---|
| `loan_amnt` | `loan_amount` | direct |
| `term` | `term` | parsed: `" 36 months"` → `36` |
| `int_rate` | `int_rate` | parsed: `"12.45%"` → `12.45` |
| `grade` | `grade` | direct |
| `emp_length` | `emp_length` | `"10+ years"` → `10`, `"< 1 year"` → `0`, `"n/a"`/NaN → `0` |
| `home_ownership` | `home_ownership` | direct |
| `annual_inc` | `annual_income` | direct |
| `purpose` | `purpose` | direct |
| `dti` | `dti` | direct |
| `fico_range_low`, `fico_range_high` | `fico` | mean |
| `open_acc` | `open_acc` | direct |
| `revol_util` | `revol_util` | parsed: `"45.2%"` → `45.2`, `NaN` preserved |
| `issue_d` | `issue_d` | parsed: `"Dec-2014"` → `2014-12-01` |
| `loan_status` | `default` | `Charged Off` / `Default` / `Late (31-120 days)` → `1`; rest → `0` |
| (none) | `noise_1`, `noise_2` | zero-filled |
| (none) | `protected_group` | `"unknown"` |

The `tests/test_lendingclub_swap.py` suite validates every step on a 6-row fixture, so you can verify the loader works without downloading the real ~1 GB dataset.

## What this project demonstrates

- **Defensible metric choice.** PR-AUC over accuracy under 18% prevalence; cost-aware threshold over arbitrary 0.5.
- **Leakage discipline.** All preprocessing fit on the training fold via `Pipeline`; a unit test enforces it structurally.
- **Sanity-checkable interpretability.** Synthetic noise features are designed to have zero true effect; mean |SHAP| confirms they sit ~1% of the dominant feature's contribution.
- **Honest trade-off.** The linear baseline is essentially tied with tuned LightGBM here because the synthetic DGP is additive — discussed openly rather than papered over.
- **Operational hygiene.** Cost-matrix sensitivity, calibration audit (Platt + isotonic, both held-out-half and CV-on-train), PSI drift monitoring on simulated regime shifts, label-delay rolling-performance monitor, fairness audit with the four-fifths rule and the inherent demographic-parity / equal-opportunity / predictive-parity trade-off spelled out, and **temporal validation** showing the 6–7pp PR-AUC gap between random and out-of-time splits.
- **Real-data swap path.** `load_data(source="lendingclub")` reads any LC accepted-loans CSV from `data/raw/` and emits the same 17-column schema; a fixture-based test exercises every parsing edge case so you can verify the loader without downloading the ~1 GB dataset.
