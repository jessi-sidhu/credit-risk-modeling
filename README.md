# Loan Default Prediction

End-to-end binary-classification project: given a loan application, predict the probability of default. Three models (logistic regression, random forest, tuned LightGBM) compared on PR-AUC, with cost-aware threshold selection (FN:FP = 5:1) and SHAP interpretability against a known synthetic data-generating process.

The pipeline is built on a synthetic Lending-Club-style dataset for reproducibility, with a single swap point (`src/data.load_data`) so a real Lending Club CSV dropped into `data/raw/` runs through the same code.

**Headline result.** Tuned LightGBM hits PR-AUC 0.74 / ROC-AUC 0.92 on the held-out test set; moving from the default 0.5 threshold to the cost-optimal 0.14 cuts expected loss by **33.9%** under the assumed 5:1 FN:FP ratio. Full numbers and discussion in [reports/report.md](reports/report.md).

## Quickstart

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
pytest                                     # 16 tests — sanity, leakage guard, threshold selection
python -m src.train --quick                # 5k rows, 10 tuning iters — under 90s
python -m src.train                        # full run — ~7 min on a laptop
```

To regenerate the notebooks after training:

```bash
jupyter nbconvert --to notebook --execute notebooks/01_eda.ipynb --inplace
jupyter nbconvert --to notebook --execute notebooks/02_modeling.ipynb --inplace
jupyter nbconvert --to notebook --execute notebooks/03_interpretation.ipynb --inplace
```

The modeling notebook caches artifacts to `reports/artifacts/`; re-runs after the first full train take seconds.

## Layout

```
src/
  synth.py        synthetic data generator (calibrated 18% prevalence)
  data.py         load_data() — single swap point for real LC data
  features.py     ColumnTransformer + four engineered ratio features
  models.py       Pipeline factories: LR, RF, LightGBM
  evaluate.py     PR-AUC, ROC-AUC, cost-aware threshold sweep
  train.py        train + RandomizedSearchCV on LightGBM
  interpret.py    SHAP wrapper for the (preprocessor, classifier) pipeline
  errors.py       per-subgroup FN/FP analysis with FNR-lift
notebooks/
  01_eda.ipynb              class balance, distributions, correlations
  02_modeling.ipynb         results table, PR curves, cost-vs-threshold
  03_interpretation.ipynb   SHAP global + local + error subgroups
reports/
  report.md       writeup (problem, methods, results, limitations, future work)
  figures/        PNGs (regenerated by notebooks)
  artifacts/      trained model + cached predictions (regenerated by training)
tests/            sanity, leakage guard, threshold selection
```

## What this project demonstrates

- **Defensible metric choice.** PR-AUC over accuracy under 18% prevalence; cost-aware threshold over arbitrary 0.5.
- **Leakage discipline.** All preprocessing fit on the training fold via `Pipeline`; a unit test enforces it structurally.
- **Sanity-checkable interpretability.** Synthetic noise features are designed to have zero true effect; mean |SHAP| confirms they sit ~1% of the dominant feature's contribution.
- **Honest trade-off.** The linear baseline is essentially tied with tuned LightGBM here, because the synthetic DGP is additive — discussed openly rather than papered over.
